== PROJECT CONTEXT — NỘI THẤT TIỆN LỢI ==
Framework: Next.js 15 + React 19 + TypeScript
Database: PostgreSQL + Prisma ORM
UI: Bootstrap 5 + Bootstrap Icons
Validation: Zod
Auth: JWT cookie-based
3-layer: Repository → Service → Route
Folder:
  src/admin/features/{module}/ — Table, Form, Filters (client components)
  src/admin/layout/{module}s/ — Page server components
  src/admin/api/{module}/ — API routes
  src/server/repositories/ — Prisma queries
  src/server/services/ — Business logic + Zod
  src/server/validators/ — Zod schemas
  src/lib/ — types.ts, constants.ts, utils.ts
  src/server/errors.ts — AppError subclasses

Trả về markdown thuần, đúng cấu trúc 6 section bên dưới.

# PHÂN TÍCH UI — [TÊN MÀN HÌNH]

## 1. TÓM TẮT CHUNG
- Mô tả màn hình, mục đích nghiệp vụ, ai dùng, module nào
- Thành phần nhìn thấy trong ảnh (liệt kê chi tiết)
- Assumptions (suy luận, không thấy rõ)
- Câu hỏi cần confirm trước khi code

## 2. PROMPT CHO DATABASE ARCHITECT
Bọc trong code block. Nội dung:
- Phân tích dữ liệu từ UI: từng field + type + suy luận
- Quan hệ dữ liệu giữa models
- Yêu cầu: Prisma schema, giải thích field, index, query pattern, edge case
- Tech: PostgreSQL + Prisma ORM

## 3. PROMPT CHO BACKEND DEVELOPER
Bọc trong code block. Nội dung:
- API endpoints: method, path, input, output cho từng action nhìn thấy
- Layer 1 Validator (Zod): schema + rule từng field
- Layer 2 Repository: methods cần implement
- Layer 3 Service: business logic
- Business rules, error cases
- Files cần tạo/sửa

## 4. PROMPT CHO FRONTEND DEVELOPER
Bọc trong code block. Nội dung:
- Layout tổng thể: bố cục, màu sắc
- Từng thành phần: bảng (cột, data), bộ lọc (input type, options), action buttons (icon, màu, hành vi)
- Components cần tạo (tên, server/client)
- State management
- API calls từ UI (khi load, khi action)
- UX: loading, error, empty, success, confirm dialog
- Files cần tạo/sửa

## 5. PROMPT CHO QA ENGINEER
Bọc trong code block. Nội dung:
- Happy path: từng usecase với bước + expected
- Edge cases
- Validation cases: từng field sai → expected error
- Permission & security
- Performance
- Manual QA checklist: □ action → expected result
- Regression targets

## 6. CHECKLIST TRIỂN KHAI
- Thứ tự thực hiện (7 bước)
- Files sẽ thay đổi (theo layer)
- Risk level + lý do
