# Unit Test Checklist — Mỗi Tính Năng

> Checklist này áp dụng cho **mỗi module/feature mới** khi viết code. Đọc checklist TRƯỚC KHI code để tránh bugs phổ biến, sau đó viết tests theo checklist.

## Cách dùng

1. Trước khi code feature mới → đọc checklist tương ứng
2. Sau khi code xong → tick checklist và viết tests tương ứng
3. Chạy `npm test` trước khi commit

---

## 1. Validator Tests (`src/server/validators/__tests__/`)

### Bắt buộc cho mọi validator

```
□ Test: valid data → success
□ Test: empty data → success (nullable fields)
□ Test: missing required field → fail
□ Test: null values for optional fields → success
□ Test: default values applied correctly
□ Test: each field boundary value (min/max length)
□ Test: invalid format (VD: bad email, bad slug)
□ Test: type coercion (VD: string → number với z.coerce)
□ Test: extra/unknown fields ignored → success
```

### Theo loại field

| Field type | Test cases |
|---|---|
| `z.string().min(1)` | empty string → fail, whitespace only → ? |
| `z.string().email()` | "not-email" → fail, "a@b.c" → success |
| `z.string().regex(slugRegex)` | "has space" → fail, "valid-slug" → success |
| `z.coerce.number()` | "123" → 123 (number), "abc" → fail |
| `z.boolean()` | true → true, false → false, "1" → ? |
| `z.enum([...])` | invalid enum value → fail |
| `z.optional()` | undefined → undefined (không fail) |
| `z.nullable()` | null → null (không fail) |
| `z.number().min(0)` | -1 → fail, 0 → success |
| `z.number().max(n)` | n+1 → fail, n → success |

### Template test file

```typescript
// src/server/validators/__tests__/{name}-validator.test.ts
import { describe, it, expect } from 'vitest';
import { validate{Name}, type {Name}Input } from '../{name}.validator';

describe('{Name} Validator', () => {
  describe('valid data', () => {
    it('validates valid data', () => {
      const data = { /* valid fields */ };
      const result = validate{Name}(data);
      expect(result.success).toBe(true);
    });
  });

  describe('required fields', () => {
    it('rejects missing required field', () => {
      const data = { /* missing name */ };
      const result = validate{Name}(data);
      expect(result.success).toBe(false);
    });
  });

  describe('optional fields', () => {
    it('accepts empty data', () => {
      const result = validate{Name}({});
      expect(result.success).toBe(true);
    });

    it('accepts null for optional fields', () => {
      const data = { name: 'Test', description: null };
      const result = validate{Name}(data);
      expect(result.success).toBe(true);
    });
  });

  describe('field validation', () => {
    // Test each field
  });

  describe('defaults', () => {
    it('applies default values', () => {
      const result = validate{Name}({});
      expect(result.success).toBe(true);
      if (result.success) {
        expect(result.data.isActive).toBe(true);
      }
    });
  });
});
```

## 2. Service Tests (`src/server/services/__tests__/`)

### Bắt buộc cho mọi service

```
□ Test: getAll() → returns array
□ Test: getById() with valid id → returns item
□ Test: getById() with invalid id → throws NotFoundError
□ Test: getBySlug() with valid slug → returns item
□ Test: getBySlug() with invalid slug → returns null

□ Test: create() with valid data → returns created item
□ Test: create() with invalid data → throws ValidationError
□ Test: create() with duplicate slug → throws DuplicateError
□ Test: create() without slug → auto-generates slug from name
□ Test: create() without code → auto-generates code from name

□ Test: update() with valid data → returns updated item
□ Test: update() with invalid id → throws NotFoundError
□ Test: update() with invalid data → throws ValidationError
□ Test: update() slug to existing value → throws DuplicateError
□ Test: update() name without slug → auto-generates slug

□ Test: delete() with valid id → returns deleted item
□ Test: delete() with invalid id → throws NotFoundError
□ Test: delete() with children (nếu có parent-child) → throws ConflictError
□ Test: delete() with related items → throws ConflictError
```

### Mock strategy

```typescript
// src/server/services/__tests__/{name}.service.test.ts
import { describe, it, expect, vi, beforeEach } from 'vitest';

// Mock repository
vi.mock('@/server/repositories/{name}.repository', () => ({
  {name}Repository: {
    findAll: vi.fn(),
    findById: vi.fn(),
    findBySlug: vi.fn(),
    create: vi.fn(),
    update: vi.fn(),
    delete: vi.fn(),
  },
}));

// Mock utils
vi.mock('@/lib/utils', () => ({
  createSlug: vi.fn((text) => text.toLowerCase().replace(/\s+/g, '-')),
}));

import { {name}Service } from '../{name}.service';
import { {name}Repository } from '@/server/repositories/{name}.repository';
import { NotFoundError, ValidationError, DuplicateError, ConflictError } from '@/server/errors';

describe('{Name} Service', () => {
  beforeEach(() => { vi.clearAllMocks(); });

  describe('getAll', () => {
    it('returns all items', async () => {
      const mockItems = [{ id: '1', name: 'Item 1' }];
      {name}Repository.findAll.mockResolvedValue(mockItems);
      const result = await {name}Service.getAll();
      expect(result).toEqual(mockItems);
    });
  });

  describe('getById', () => {
    it('returns item for valid id', async () => {
      const mockItem = { id: '1', name: 'Item 1' };
      {name}Repository.findById.mockResolvedValue(mockItem);
      const result = await {name}Service.getById('1');
      expect(result).toEqual(mockItem);
    });

    it('throws NotFoundError for invalid id', async () => {
      {name}Repository.findById.mockResolvedValue(null);
      await expect({name}Service.getById('invalid')).rejects.toThrow(NotFoundError);
    });
  });

  describe('create', () => {
    it('creates item with valid data', async () => {
      const input = { name: 'New Item', slug: 'new-item' };
      const mockCreated = { id: '1', ...input };
      {name}Repository.create.mockResolvedValue(mockCreated);
      {name}Repository.findBySlug.mockResolvedValue(null);
      const result = await {name}Service.create(input);
      expect(result).toEqual(mockCreated);
    });

    it('throws ValidationError for invalid data', async () => {
      // mock validator fail
      await expect({name}Service.create({})).rejects.toThrow(ValidationError);
    });

    it('throws DuplicateError for duplicate slug', async () => {
      {name}Repository.findBySlug.mockResolvedValue({ id: 'existing' });
      await expect({name}Service.create({ name: 'Test', slug: 'test' }))
        .rejects.toThrow(DuplicateError);
    });

    it('auto-generates slug if missing', async () => {
      {name}Repository.create.mockResolvedValue({ id: '1', name: 'Test Item', slug: 'test-item' });
      {name}Repository.findBySlug.mockResolvedValue(null);
      const result = await {name}Service.create({ name: 'Test Item' });
      expect({name}Repository.create).toHaveBeenCalledWith(
        expect.objectContaining({ slug: 'test-item' })
      );
    });
  });
});
```

## 3. Repository Tests (`src/server/repositories/__tests__/`)

### Bắt buộc

```
□ Mock Prisma trước khi test
□ Test: findAll() returns array
□ Test: findAll(onlyActive=true) filters correctly
□ Test: findById() returns item
□ Test: findById() returns null for invalid id
□ Test: findBySlug() returns item
□ Test: findBySlug() returns null for invalid slug
□ Test: create() returns created item with all fields
□ Test: update() returns updated item
□ Test: delete() returns deleted item
□ Test: select includes only needed fields (verify no over-fetching)
```

### Mock Prisma pattern

```typescript
// src/server/repositories/__tests__/{name}.repository.test.ts
import { describe, it, expect, vi, beforeEach } from 'vitest';

const mockPrismaClient = {
  {model}: {
    findMany: vi.fn(),
    findUnique: vi.fn(),
    create: vi.fn(),
    update: vi.fn(),
    delete: vi.fn(),
    count: vi.fn(),
  },
};

vi.mock('@/lib/prisma', () => ({
  prisma: mockPrismaClient,
}));

// Import AFTER mock
import { {name}Repository } from '../{name}.repository';

describe('{Name} Repository', () => {
  beforeEach(() => { vi.clearAllMocks(); });

  describe('findAll', () => {
    it('returns all items', async () => {
      const mockItems = [{ id: '1', name: 'Item' }];
      mockPrismaClient.{model}.findMany.mockResolvedValue(mockItems);
      const result = await {name}Repository.findAll();
      expect(result).toEqual(mockItems);
      expect(mockPrismaClient.{model}.findMany).toHaveBeenCalledWith({
        where: undefined,
        select: expect.any(Object),
        orderBy: expect.any(Object),
      });
    });

    it('filters by isActive when onlyActive=true', async () => {
      mockPrismaClient.{model}.findMany.mockResolvedValue([]);
      await {name}Repository.findAll(true);
      expect(mockPrismaClient.{model}.findMany).toHaveBeenCalledWith({
        where: { isActive: true },
        select: expect.any(Object),
        orderBy: expect.any(Object),
      });
    });
  });

  describe('findById', () => {
    it('returns item for valid id', async () => {
      const mockItem = { id: '1', name: 'Item' };
      mockPrismaClient.{model}.findUnique.mockResolvedValue(mockItem);
      const result = await {name}Repository.findById('1');
      expect(result).toEqual(mockItem);
    });

    it('returns null for invalid id', async () => {
      mockPrismaClient.{model}.findUnique.mockResolvedValue(null);
      const result = await {name}Repository.findById('invalid');
      expect(result).toBeNull();
    });
  });

  describe('create', () => {
    it('creates and returns item', async () => {
      const input = { name: 'New Item' };
      const mockCreated = { id: '1', ...input };
      mockPrismaClient.{model}.create.mockResolvedValue(mockCreated);
      const result = await {name}Repository.create(input);
      expect(result).toEqual(mockCreated);
      expect(mockPrismaClient.{model}.create).toHaveBeenCalledWith({ data: input });
    });
  });
});
```

## 4. API Route Tests (`src/admin/api/__tests__/`)

### Bắt buộc

```
□ Mock service layer (không mock DB)
□ Test: GET → 200 + { success: true, data: [...] }
□ Test: GET with empty data → 200 + { success: true, data: [] }
□ Test: POST valid → 201 + { success: true, data: {...} }
□ Test: POST invalid → 422 + { success: false, error: ... }
□ Test: POST duplicate → 409 + { success: false, error: ... }
□ Test: POST missing required → 422 + { success: false, error: ... }
□ Test: GET [id] → 200 + { success: true, data: {...} }
□ Test: GET [id] not found → 404 + { success: false, error: ... }
□ Test: PUT [id] valid → 200 + { success: true, data: {...} }
□ Test: PUT [id] not found → 404 + { success: false, error: ... }
□ Test: DELETE [id] → 200 + { success: true }
□ Test: DELETE [id] not found → 404 + { success: false, error: ... }
□ Test: DELETE [id] conflict → 409 + { success: false, error: ... }
```

### Mock strategy

```typescript
// src/admin/api/__tests__/{name}.route.test.ts
import { describe, it, expect, vi, beforeEach } from 'vitest';

// Mock service
vi.mock('@/server/services/{name}.service', () => ({
  {name}Service: {
    getAll: vi.fn(),
    getById: vi.fn(),
    create: vi.fn(),
    update: vi.fn(),
    delete: vi.fn(),
  },
}));

vi.mock('@/server/errors', () => ({
  NotFoundError: class extends Error {
    constructor() { super('Not found'); this.name = 'NotFoundError'; }
  },
  ValidationError: class extends Error {
    constructor(msg: string) { super(msg); this.name = 'ValidationError'; }
  },
  DuplicateError: class extends Error {
    constructor() { super('Duplicate'); this.name = 'DuplicateError'; }
  },
}));

import { GET, POST } from '../route';
import { NextRequest } from 'next/server';
import { {name}Service } from '@/server/services/{name}.service';
import { NotFoundError, ValidationError, DuplicateError } from '@/server/errors';

describe('GET /admin/api/{name}', () => {
  beforeEach(() => { vi.clearAllMocks(); });

  it('returns 200 with success: true', async () => {
    {name}Service.getAll.mockResolvedValue([{ id: '1', name: 'Item' }]);
    const res = await GET();
    expect(res.status).toBe(200);
    const json = await res.json();
    expect(json).toEqual({ success: true, data: [{ id: '1', name: 'Item' }] });
  });
});

describe('POST /admin/api/{name}', () => {
  it('returns 201 on success', async () => {
    {name}Service.create.mockResolvedValue({ id: '1', name: 'New' });
    const req = new NextRequest('http://localhost/admin/api/{name}', {
      method: 'POST',
      body: JSON.stringify({ name: 'New' }),
      headers: { 'Content-Type': 'application/json' },
    });
    const res = await POST(req);
    expect(res.status).toBe(201);
    const json = await res.json();
    expect(json.success).toBe(true);
  });

  it('returns 422 on validation error', async () => {
    {name}Service.create.mockRejectedValue(new ValidationError('Lỗi'));
    const req = new NextRequest('http://localhost/admin/api/{name}', {
      method: 'POST',
      body: JSON.stringify({}),
      headers: { 'Content-Type': 'application/json' },
    });
    const res = await POST(req);
    expect(res.status).toBe(422);
  });

  it('returns 409 on duplicate error', async () => {
    {name}Service.create.mockRejectedValue(new DuplicateError());
    const req = new NextRequest('http://localhost/admin/api/{name}', {
      method: 'POST',
      body: JSON.stringify({ name: 'Test', slug: 'test' }),
      headers: { 'Content-Type': 'application/json' },
    });
    const res = await POST(req);
    expect(res.status).toBe(409);
  });
});
```

## 6. Bug Patterns Phổ Biến — Checklist Trước Khi Code

### Validator bugs
```
□ Quên z.string().min(1) cho required string → empty string được chấp nhận
□ z.boolean() không xử lý đúng khi nhận string "true"/"false"
□ z.coerce.number() chấp nhận "abc" → NaN
□ z.enum() không handle case-insensitive
□ Thiếu .nullable() cho fields cho phép null
□ Thiếu .optional() cho fields không bắt buộc
□ Regex slug không khớp với createSlug() utility
```

### Service bugs
```
□ Quên auto-generate slug khi input không có
□ Quên auto-generate code khi input không có
□ Không check duplicate trước create
□ Không check duplicate khi update slug
□ Không throw NotFoundError khi findById/null
□ Không validation trước khi gọi repository
□ Quên validate result từ Zod (forgot .success check)
□ Upsert nhiều bảng không dùng $transaction
□ Không handle Prisma unique constraint error → để crash
```

### Repository bugs
```
□ Select * (lấy thừa field) → security/performance
□ Quên where clause cho findAll(onlyActive)
□ Không handle null từ findUnique → trả về null gây crash ở service
□ create/update không include relations cần thiết
□ Quên unique constraint → Prisma throw lỗi không friendly
□ update với partial data không đúng cách
```

### API Route bugs
```
□ Quên try/catch → unhandled promise rejection crash server
□ Không parseAppError → error không format đúng
□ HTTP status code sai (200 thay vì 201, 500 thay vì 404)
□ Không handle missing body → request.json() crash
□ Quên headers { 'Content-Type': 'application/json' }
□ Response không kèm success flag
□ Error message hardcoded tiếng Anh
```

### Form / Client Component bugs
```
□ Quên 'use client' directive
□ Form page không dùng dynamic import → SSR crash với CKEditor
□ Submit không handle error → silent fail
□ router.push() mà không có router.refresh() → data không cập nhật
□ Validation client-side khác với validator (VD: form chấp nhận empty nhưng Zod reject)
□ Không clear errors khi user bắt đầu sửa lại
□ Checkbox không cast đúng kiểu (checked → boolean)
□ Quên noValidate trên form → browser native validation conflict
```

### Database / Prisma bugs
```
□ Schema field type không khớp với TypeScript type
□ Quên chạy prisma generate sau khi đổi schema
□ Migration conflict khi nhiều người cùng dev
□ Nullable field không có default → insert fail
□ Unique constraint không được handle ở service → Prisma error leak ra ngoài
```
