# 🖼️ AUTO GENERATE TASK TỪ ẢNH UI

## 🎯 Mục tiêu
Biến ảnh chụp giao diện UI, screenshot, mockup hoặc wireframe thành **task kỹ thuật chuẩn** để Claude Code có thể đọc, phân tích và implement đúng theo project.

Hệ này dùng khi bà chủ có:
- ảnh màn hình thật
- ảnh mockup từ Figma/Canva
- ảnh chụp bug UI
- ảnh yêu cầu chỉnh sửa giao diện

---

# 🚀 Cách dùng chuẩn

## Bước 1 — Gửi ảnh UI cho Claude
Có thể gửi:
- 1 ảnh
- nhiều ảnh
- ảnh kèm mô tả ngắn

Ví dụ:
- “Đây là ảnh màn hình menu admin, hãy convert thành task kỹ thuật”
- “Đây là UI cần làm, hãy phân tích và tạo task file”

---

## Bước 2 — Dùng prompt này

```text
Bạn đang làm việc trên dự án Nội Thất Tiện Lợi.

Nhiệm vụ:
Phân tích ảnh UI tôi cung cấp và tự động chuyển nó thành task kỹ thuật để Claude Code có thể implement.

Yêu cầu:
1. Nhận diện màn hình/chức năng từ ảnh
2. Tóm tắt mục tiêu của UI
3. Xác định module gần nhất trong project
4. Suy luận các thay đổi cần có ở:
   - UI
   - API
   - service
   - repository
   - validator
   - database (nếu có)
5. Tạo task file theo format TASK_TEMPLATE.ts
6. Thêm:
   - business rules
   - acceptance criteria
   - related files gợi ý
   - risk level
   - rollback plan
   - regression targets
   - manual QA checklist

Ràng buộc:
- Không code ngay
- Không giả định quá đà nếu ảnh không đủ rõ
- Nếu có điểm chưa chắc chắn, ghi rõ là assumption
- Ưu tiên sửa tối thiểu theo convention hiện có

Output phải gồm:
1. UI Summary
2. Assumptions
3. Related Module Guess
4. Affected Layers
5. Technical Task File
6. Risks
7. Manual QA Checklist
```

# 🧠 Flow phân tích ảnh UI

Claude phải làm theo thứ tự này:

## 1. Nhìn ảnh
- xác định đây là màn hình nào
- xác định hành động người dùng có thể làm
- xác định dữ liệu nào đang hiển thị
- xác định nút, form, filter, table, action icons

## 2. Suy ra yêu cầu nghiệp vụ
Ví dụ:
- có dropdown mới → có thể cần filter/query param
- có cột mới trong table → có thể cần update select/repository
- có nút action mới → có thể cần route/link mới
- có field form mới → có thể cần validator/service/db update

## 3. Map sang kiến trúc project
- UI component nào sẽ đổi
- page nào sẽ đổi
- service/repository nào có thể ảnh hưởng
- có cần API mới hay không
- có cần migration hay không

## 4. Sinh task file
Task phải rõ để Claude Code dùng tiếp mà không cần đọc lại ảnh nhiều lần

# 📄 Output mẫu mong muốn

## 1. UI Summary
- Đây là màn hình quản lý menu trong admin
- Có bảng danh sách
- Có filter/search
- Có action icons
- Thiếu filter theo loại menu

## 2. Assumptions
- Giả định `menuTypeId` đã tồn tại trong database
- Giả định project đã có module menu
- Giả định action icon mới chỉ là điều hướng, không mở modal

## 3. Related Module Guess
- module: `menu`
- reference module gần nhất: `category`

## 4. Affected Layers
- repository
- service
- admin ui
