# 🚀 Cách dùng Claude Code (Flow Chuẩn)

## 🧠 Tổng quan
Flow chuẩn giúp Claude Code hoạt động như một Senior Developer:
- Không code ngay
- Hiểu project trước
- Có plan rõ ràng
- Code đúng convention
- Tự review và test

---

## 🔥 Bước 1: Gọi Claude đọc skill + task

```bash
Please read these files first:
- ./skill-noithat-dev.ts
- ./unit-test-checklist.md
- ./tasks/menu_feature_task.ts

Then:
1. Analyze project structure
2. Identify related files
3. Summarize current logic
4. Show implementation plan
5. Show risks
DO NOT WRITE CODE YET
```

---

## ⚙️ Bước 2: Xác nhận trước khi code

```bash
Proceed with implementation.
Follow the skill strictly.
```

---

## 💻 Bước 3: Claude bắt đầu code

Claude sẽ:
- Sửa đúng file cần thiết
- Không phá cấu trúc
- Tuân thủ convention project

---

## 🧪 Bước 4: Kiểm tra & test

```bash
Now:
- verify all changes
- check edge cases
- follow unit-test-checklist.md
- list test cases
```

---

## 🔍 Bước 5: Review code

```bash
Review the code:
- check naming convention
- check architecture
- check unnecessary changes
- check potential bugs
```

---

## 🎯 Flow nhanh (dùng hằng ngày)

```bash
Read skill + task → analyze → plan → risk → then code → then review → then test
```

---

## 🧠 Các mode quan trọng

### 🧪 Debug mode

```bash
Analyze this feature:
- find bugs
- find missing logic
- suggest fix
DO NOT CODE
```

### 🚀 Feature mode

```bash
Implement this task:
- follow skill
- minimal changes
- explain each file modified
```

### 🛑 Safe mode

```bash
Only modify files directly related.
Do not:
- change global config
- add libraries
- refactor unrelated code
```

---

## 💡 Tips quan trọng

```bash
First analyze → then plan → then code
```

```bash
Do not assume. Ask if anything is unclear.
```
